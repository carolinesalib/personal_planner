# App Framework and Philosophy

This document captures the core methodology behind the app. It should inform all copy, UX, and feature decisions.

## The daily flow

1. **Evening planning** — Plan the next day before going to sleep, so you wake up already focused and intentional.
2. **Today's Plan** — The default landing page. Return to it many times throughout the day as an anchor for your focus.
3. **Gratitude** — Done during the planning session, as part of closing out the day.
4. **Shoulds** — A holding place for "I should do X" thoughts. Write them here the moment they surface, then consult this list when planning the next day.
5. **(Future) Week / Quarter / Year planning** — Will draw from Shoulds and daily planners. Users can pull saved items as suggestions when building longer-horizon plans.

## Core values

### Planning is intention, not perfection
Planning is always about setting intentions. It is completely okay not to complete everything — what matters is that you are setting intentions and moving toward them. We go for progress, not perfection. The app should never make users feel bad for incomplete days.

### Priorities: recommend 5, never overwhelm
For daily priorities, we recommend a maximum of 5. The purpose is focus, not volume. A shorter list done well beats a long list left undone.

### Gratitude: 3 daily, but skip on busy days
We recommend 3 gratitude entries per planning session, but it is always okay to skip on busy days. The habit should feel like a gift, not a chore.

### Shoulds are a relief valve, not a to-do list
The Shoulds page is not where you go to work. It is where you go to stop carrying things in your head. Write it down, let go, and decide during planning whether it is the right time to act on it.
